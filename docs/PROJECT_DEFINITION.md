# Project Definition

## Product Name

FarsamoTech Repair Hub

## Team Name

XeelTech Solutions

## Purpose

FarsamoTech Repair Hub is a centralized platform for managing computer repair and maintenance services at SIMAD University. It supports students, lecturers, technicians, and administrators through a structured repair workflow, transparent status tracking, and data-driven reporting.

## Problem Statement

SIMAD University's current Computer Maintenance Week process relies on manual registration, paper records, and Excel sheets. This creates operational issues:

- No centralized repair database
- No reliable repair status tracking
- No technician performance monitoring
- No historical repair record per student, lecturer, or device
- No analytics for recurring computer issues
- High risk of data loss
- Limited communication between request owners and technicians

## Product Goals

- Digitize repair request submission and tracking
- Provide a clear repair journey for every device
- Help technicians manage assigned tickets and repair notes
- Give administrators visibility into workload, performance, and trends
- Preserve repair history for long-term institutional use
- Establish a scalable foundation for future mobile, AI, inventory, and notification features

## Primary Users

### Student

- Register online
- Submit repair requests
- Track repair progress
- View repair history

### Lecturer

- Submit repair requests
- Track repair progress
- Access relevant repair reports

### Technician

- View assigned tickets
- Update repair status
- Add diagnosis and repair notes
- Record working hours
- Review daily tasks and performance metrics

### Admin

- Manage users
- Assign technicians
- Monitor repair progress
- View reports and analytics
- Review technician performance

## MVP Scope

The MVP should include:

- User registration and authentication
- Role-based access for student, lecturer, technician, and admin users
- Device registration
- Repair request creation
- Ticket ID and QR code generation
- Repair journey status tracking
- Technician assignment
- Repair logs and notes
- Dashboard notifications
- Admin reports and analytics
- Repair history per user and device

## Future Scope

Phase 2 may include:

- Mobile application
- AI issue classification
- SIMAD student portal integration
- Inventory and spare parts management
- SMS and WhatsApp notifications
- Advanced technician ranking and recognition
